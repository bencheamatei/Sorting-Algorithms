## 1. Overview

In this project, you will work in teams of 1–3 students. The goal is to implement multiple sorting algorithms and evaluate their performance on a diverse set of test cases.

---

## 2. Requirements

You are required to implement the following sorting algorithms:

- Heapsort
- Quicksort
- Mergesort
- Introsort
- Patience Sorting
- Radix Sort
- Bubble Sort
- Insertion Sort
- Gnome Sort

---

## 3. Testing

In addition to implementing the algorithms, you must design a diverse set of tests. The goal is to highlight the strengths and weaknesses of each algorithm (for example, best/worst case tests for these algorithms).

### 3.1 Test Categories

There are two types of tests:

- **Small tests:** N = 20,000 (intended mainly for O(n²) algorithms).
- **Big tests:** N = 10,000,000 (intended for efficient algorithms).

Each test consists of sorting N integers of 32 bits.

---

## 4. Input / Output Requirements

Since input speed can significantly affect benchmarking results, you must use the following input template and implement only the `custom_sort` function:

```cpp
int main() {
    std::ios::sync_with_stdio(false);
    std::cin.tie(nullptr);
    int n;
    std::cin >> n;
    std::vector<int> a(n);
    for (int& x : a) {
        std::cin >> x;
    }
    custom_sort(a);
    assert(std::ranges::is_sorted(a));
    return 0;
}
```

## 5. Submission Requirements

All files must be uploaded to the Polygon platform.

### 5.1 Source Files Naming

Source files must follow the format:

```
algorithm_team[a].cpp
```

**Example:**
```
quicksort_team1.cpp
```

### 5.2 Test Files Naming

Test files must follow the format:

```
type_(small/big)_team_id.in
```

And corresponding outputs:

```
type_(small/big)_team_id.ok
```

**Examples:**
- `small_team1_01.in`
- `big_team2_03.in`

At the end of the project, you will be asked to submit the archive generated by Polygon, along with a document describing how each test case (or batch of test cases) was designed.
